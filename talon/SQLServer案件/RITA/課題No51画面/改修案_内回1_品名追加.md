# 【課題No51】「内回1」工程に品名追加機能（品名ボタン方式）を追加

## 背景
IVH内製基板で「内回1」工程に品名が入ってこない。4層以上で「内回」には L2,L3 などが入るが、「内回1」は空欄になっている。「コメント」工程と同様の品名ボタンを「内回1」にも追加し、明細各層に品名を行ごとに設定できるようにする。

- L2～L19 の明細行に品名ボタンを設置する
- L0 の列には親品名を自動設定（SIYO2002 で親品名設定は改修済み）
- 6/17 に資料送付済み

## 現状（課題No51の構成）
- ブロック1：ホルダー（`DATA0001` 固定1行、`SIYO_SEIZO_JKN_HEADER`）
- ブロック2：明細（`SIYO_SEIZO_JKN_MEISAI2`、`LAYER = 'L' + (GYOSU+1)` で L2,L3… を表現、`DATA01`=備考）
- ブロック3：プログラム一覧（`SIYO_SEIZO_JKN_MEISAI`）
- ブロック4：備考一覧
- 登録処理後JS（`SQL/登録処理後/処理.js`）：固定1行＋各ブロックのinsertのみ。品名を扱う処理は無し。

## 参照実装（コメント工程の品名ボタン）
クライアントJS（`品名設定/設定.js`）が品名ボタンの本体。行ごとにボタンから呼び出す。

```js
// 品名（HIN_NM）を押下行へコピー
function setHinNm(targetObj) {
    var objHinNm = getElementByName("1_HIN_NM");   // ブロック1の品名項目
    if (!objHinNm) return;
    targetObj.value = objHinNm.value;              // targetObj = 押下した行の入力欄
}
// 仕様No（先頭2文字を除去）を押下行へコピー
function setSiyoNo(targetObj) {
    var objSiyosyoNo = getElementByName("1_SIYOSYO_NO");
    if (!objSiyosyoNo) return;
    targetObj.value = objSiyosyoNo.value.substring(2);
}
```

ポイント：`getElementByName("1_HIN_NM")` の `1_` はブロック1のプレフィックス。コメント工程ではブロック1に品名(`HIN_NM`)・仕様No(`SIYOSYO_NO`)がJS参照可能な項目として置かれている。内回1にはこれらが無いので追加が必要。

## 改修方針
コメント工程と同じ「行ごとのJavaScript呼出ボタン＋クライアントJS」で品名をコピーする方式を踏襲する。

## やること

### ① ブロック1に品名・仕様No項目を用意（JS参照用）
- 内回1のブロック1へ `HIN_NM`（品名）と `SIYOSYO_NO`（仕様No）を取得項目として追加し、項目名を `1_HIN_NM` / `1_SIYOSYO_NO` にする。
- 画面に出さない場合も、非表示項目をJS参照可能にする設定を入れる（マニュアル 3.2.8）。
- 取得元は `SIYOSYO_MAIN_WORK`（`HIN_NM` / `SIYOSYO_NO`）。1ブロック.sql を改修。

### ② クライアントJS（品名設定）を内回1へ組み込む
- `setHinNm` / `setSiyoNo` を内回1機能のClient-side JSとして登録（共通化する場合は 5.12.2）。
- マニュアル：1.6.9（Client-side JS）。

### ③ ブロック2（明細 L2～L19）に品名列＋品名ボタンを設置
- ブロック2に品名の更新列を追加する（2ブロック.sql を改修）。
- 各行に JavaScript呼出ボタンを設置し、`setHinNm(targetObj)`（targetObj=その行の品名列）を呼ぶ（マニュアル 3.5.9）。
- 仕様Noも行に持たせる場合は `setSiyoNo` 用ボタン／呼出も同様に追加。

### ④ L0への親品名の自動設定
- L0 行の品名列に親品名を自動設定する。親は `IVH_OYA_HINMOKU_CD` / `IVH_OYA_SIYO_NO` / `IVH_OYA_SIYO_SEQ` で辿る（SIYO2002で親品名設定済みの値を利用）。
- 設定タイミングに応じて、追加行の初期値（3.4.7）または登録処理後JS（`処理.js`／1.6.24）で実装。

### ⑤ 登録処理後JSの改修（`SQL/登録処理後/処理.js`）
- ブロック2に追加した品名列をDB登録するよう `_insertFixedLines` ／ブロックinsert処理に品名項目を追加。

## 確認事項（業務へ）
- **子基板が2層の場合、システムは内層が無いと判断するか？** 内回1の層・品名の扱いに影響するため要確認。
- 品名ボタンで設定する値は「コメント工程と同じ品名（HIN_NM）」でよいか。仕様No（`setSiyoNo`）も内回1で必要か。
- L0自動設定のタイミング（画面起動時／確定時）の希望。

## テスト観点
- 4層以上のIVH内製基板で、内回1のL2～L19各行に品名ボタンが表示され、押下でブロック1の品名がその行にコピーされる
- L0列に親品名が自動設定される
- 品名を設定して確定 → DB（明細）に品名が登録される
- 既存のブロック1～4の登録（ホルダー・明細・プログラム・備考）にデグレが無い
- 子基板2層のケースで想定どおりの挙動（※確認事項の結論に従う）
