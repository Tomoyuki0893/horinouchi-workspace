# 【課題No60431827】子基板のとき仕上板厚の初期値を変更

## 背景
子基板（＝表面処理が「IVH内層板作成」）の場合、仕様書ヘッダーの仕上板厚の初期値を以下にしたい。親から子基板を生成したタイミングの初期値が対象。

| 項目 | 現状 | 変更後 |
|---|---|---|
| 仕上板厚（`SIAGE_ATU`） | 発注書の板厚（＝完成時の板厚）が入る | **空欄** |
| 仕上板厚公差（`SIAGE_ATU_P` / `SIAGE_ATU_M`） | 発注書の公差 | **±0.1**（P=0.1 / M=0.1） |

## 対象
`初期値設定処理/JUST_DB_API.js` の `_insertSiyosyoMainWork()`。仕上板厚まわりの3行（現状）：

```js
putMapValue(map, 'SIAGE_ATU',   nvlNumber(mitsumoriRec ? mitsumoriRec.field_1690751537 : null)); // 仕上厚
putMapValue(map, 'SIAGE_ATU_P', nvlNumber(hacchuRec ? hacchuRec.field_1673848281 : null));       // 公差 +
putMapValue(map, 'SIAGE_ATU_M', nvlNumber(hacchuRec ? hacchuRec.field_1713766844_72 : null));    // 公差 -
```

## 改修内容
表面処理が「IVH内層板作成」のときだけ、仕上板厚を空欄・公差を±0.1で上書きする。上記3行を以下に置き換える。

```js
// 表面処理名で子基板（IVH内層板作成）かを判定
var hyomenName = getJustKomokuCode(mitsumoriRec, "field_1661158203");
var isIvhNaisou = (String(hyomenName || '').indexOf("IVH内層板作成") >= 0);

if (isIvhNaisou) {
    // 子基板：仕上板厚は空欄、公差は±0.1
    map['SIAGE_ATU'] = null;                 // 空欄（数値項目はNULLで登録）
    putMapValue(map, 'SIAGE_ATU_P', 0.1);
    putMapValue(map, 'SIAGE_ATU_M', 0.1);
} else {
    putMapValue(map, 'SIAGE_ATU',   nvlNumber(mitsumoriRec ? mitsumoriRec.field_1690751537 : null));
    putMapValue(map, 'SIAGE_ATU_P', nvlNumber(hacchuRec ? hacchuRec.field_1673848281 : null));
    putMapValue(map, 'SIAGE_ATU_M', nvlNumber(hacchuRec ? hacchuRec.field_1713766844_72 : null));
}
```

注意：
- `SIAGE_ATU` の「空欄」は数値項目のため **NULL** で登録する（`putMapValue` は null を空文字 `""` に変換してしまうので、`map['SIAGE_ATU'] = null;` と直接セット）。
- `±0.1` は P=0.1 / M=0.1 で登録。マイナス側を符号付き（-0.1）で持つ運用なら `SIAGE_ATU_M` を -0.1 にする。

## 確認事項
- 子基板の判定は「表面処理名＝IVH内層板作成」でよいか。コード（`HYOMEN_SYORI_CD`）での判定が望ましい場合は、対応コード値を確認して条件を差し替える。
- 公差マイナス側の保持方法（0.1 か -0.1 か）。
- `SIAGE_ATU` 空欄は NULL でよいか（画面表示・後続帳票で 0 を期待する箇所がないか）。

## テスト観点
- 表面処理「IVH内層板作成」で取込 → 仕上板厚が空欄、公差 P/M が 0.1 になる
- それ以外の表面処理で取込 → 従来どおり（発注書の板厚・公差が入る）
- 既存の仕様書メインワーク作成・工程順登録にデグレが無い
