# 【SIYO3R110】工程111「穴明1」を任意工程化＋登録処理後JSのゼロ除算修正

## 背景
2コア4層板などで工程111「穴明1」が必要だが、現状111は「重ね」「ミニVIA」入力が前提で確定できない。110は必須のまま、111を任意化したい。

※111任意化に伴い、必須項目の薄い青(水色)背景は消えます（マニュアル3.2.4の仕様どおり。業務了承済み）。

## 現象
- 111で「端面研磨のみ実施」だけ入力 → 確定不可
- 「重ね」「ミニVIA」に 0 入力 → 確定不可

## やること

### ① 更新必須の解除（マニュアル 3.2.4）
ブロック設定画面 ＞ 項目情報タブ を開き、工程111の「重ね」「ミニVIA」項目の **「更新必須」チェックをOFF**。

- 110側の項目は変更しない。
- 注意（3.2.4の制約）：対象項目が **PKEY だと必須OFFでも自動で必須チェックが効く**。PKEYでないことを確認。
- 注意：対象項目に **DB設定式があり結果がNULLになると必須エラー**になる。設定式の有無を確認。

### ② 入力チェックパターンの確認（マニュアル 1.6.15）
項目情報タブ／詳細設定タブで、111の「重ね」「ミニVIA」に入力チェックパターン（条件式）が設定されていないか確認。0や未入力を弾く条件があれば見直す。

### ③ JS修正（`SIYO3R110/登録処理後/処理.js` の `_updateMiniViaToMain`）
ガード条件のバグ修正（確定処理後JSは確定フローの最後に実行＝マニュアル8.6）。

```js
// 修正前（常にfalseで効かない → SOU_HITが0/nullだとゼロ除算で確定失敗の恐れ）
if (!SOU_HIT && SOU_HIT > 0) return;
// 修正後
if (!SOU_HIT || SOU_HIT <= 0) return;
```

## テスト観点
- 111で端面研磨のみ／重ね・ミニVIAに0／空 → いずれも確定OK
- 111に正常値（総ヒット>0）→ MINI_VIA が正しく計算
- SOU_HIT=0/null → 落ちずに更新スキップ
- 110は従来どおり必須が効く（デグレ確認）
- 必須を外した項目の水色背景が消えていること
