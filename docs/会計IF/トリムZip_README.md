# 会計IF 画面用 LogicDesigner トリム Zip

`intra-mart/logic-designer/TMH/im_logicdesigner-data_会計IF画面trim.zip`

TMH の BloomMaker 14 画面が実際に呼ぶフローだけに絞った im-LogicDesigner インポート用パッケージ。
**不要なフロー（画面から到達しない約400フロー）を除外**しているため、インポートしても
それらは上書きされない。

## 内容

| 項目 | トリム | 元 |
|---|---|---|
| フロー（ユニーク） | **29** | 430 |
| flowDefinitions（版込み） | 82 | 696 |
| ルート | 29 | 286 |
| フロー図 PNG | 82 | 676 |
| icon_data | 12（全保持） | 12 |
| サイズ | 2.8 MB | 41.7 MB |

- 全 29 フローは `会計IF` カテゴリ配下。サブフロー参照の不足なし（閉包検証済み）。
- `flow_route.json` は 29 ルートのみ。画面が呼ぶルートを全件収録済み（検証済み）。
- `user_definition.json` は元のまま全保持（ユーザー定義関数の参照切れ防止）。
- 構造（`flow_definition.json` / `flow_route.json` / `user_definition.json` / `flow_diagram/` / `icon_data/`）は原本と同一。

## 除外したもの

- **会計IF外の `【受注】検索`（`tmh_customer_orders` / 社内アプリ・受注出荷）**
  → 別ドメインの資産のため意図的に除外。**対象環境に既存であることが前提**。
  部品仕入単価・仕掛品出庫の2画面が受注エラー照会で使用するため、無い環境では別途用意が必要。
- 画面から到達しないその他の全フロー（バッチ/IF系など約400）。

## ⚠️ 重要：これは「絞り込み」であって「修正」ではない

トリムZipは現行フローを**そのまま**含む。[必要ロジック抽出.md](必要ロジック抽出.md) で指摘した不具合は**未修正のまま入っている**：

- 🔴 `仕掛品出庫入力データ更新`：UPDATE文の SET句カンマ抜け＋WHERE直前の余分カンマ（**SQL構文エラー**）。更新がHTTP GET。
- 🟠 `TMC買掛金不照合追加`：チェック工程が「TMC**売掛金**不照合追加入力データチェック」を参照（ドメイン取り違えの疑い）。

インポート前にこれらの修正要否を判断すること。

## インポート時の注意

- **本番前に検証テナントでテストインポート**すること。手作業でフィルタした
  パッケージのため、対象バージョンのインポータでの取り込み可否は未検証。
- テーブルが揃っていること（[テーブル・エンティティ突合.md](テーブル・エンティティ突合.md) 参照）。
  特に `processing_defect_quantities` / `specific_labor_hours` /
  `work_in_progress_issue_quantities` は entity export・DDL に無く **xlsx にのみ定義**があるため、
  DB 側にテーブルが存在するか要確認。
