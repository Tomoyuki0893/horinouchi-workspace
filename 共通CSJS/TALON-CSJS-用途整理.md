# TALON Client-side JavaScript（CSJS）用途整理

TALON公式マニュアル（機能構築マニュアル / Client-side JavaScriptコードサンプル）をもとに、
「CSJSとは何で、何のために使うのか」を整理したメモ。

---

## 1. まず大前提：TALONのJavaScriptは2種類ある

ここが混乱しやすいポイント。TALONで「JavaScript」と言うと2つあり、役割が全く違う。

| 種類 | 実行場所 | できること | できないこと |
|---|---|---|---|
| **JavaScriptエンジン** | サーバ側 | DB検索・更新、SQL発行、採番、バッチ呼出、他画面呼出など**業務ロジック** | 画面（ブラウザ）の直接操作 |
| **Client-side JavaScript（CSJS）** | ブラウザ側 | ボタンクリック/フォーカスアウト等の**画面操作・イベント処理**、画面の値を使った計算・表示制御 | **DB接続・データ受け渡しは不可** |

> CSJSは「画面の見た目・振る舞い」専門。データベースには一切触れない。
> DBが絡む処理はJavaScriptエンジン（サーバ側）の担当。

### なぜCSJSは「ピンとこない」のか
TALONは標準機能（ノーコード）で画面の大半を作れてしまう。
だからCSJSの出番は **「標準機能だけでは実現できない画面操作」だけ** に絞られる。
＝ 普段はあまり書かない → 用途がイメージしにくい、という構図。

---

## 2. CSJSの典型的な用途（これがCSJSの存在理由）

標準機能で足りない、次のような「画面まわりの細かい制御」に使う。

1. **入力チェック（バリデーション）** … 確定/検索の前に、画面の入力値を検証してNGなら止める。
2. **画面の値を使った計算・自動表示** … 単価×数量＝金額を、入力に応じてその場で表示。
3. **項目の表示制御** … 条件によって項目を非表示/活性・非活性、色を変える。
4. **二重送信防止・確認ダイアログ** … ボタン連打防止、「登録しますか？」確認。
5. **イベントに応じた処理** … 検索完了後・行追加後・帳票出力後にメッセージを出す等。

> 今回作っている共通CSJS（validation.js / format.js / date.js / ui.js / rest.js）は、
> まさに上記1〜4を「どの画面からでも呼べる部品」にまとめたもの。

---

## 3. CSJSの「設定箇所」と挙動の違い（重要）

CSJSは1か所ではなく、複数の場所に書ける。**書く場所によって展開のされ方が違う**ので要注意。

### (A) 機能設定 ＞ アプリケーションタブ ＞ Client-side JavaScript
- 画面の `<script>` 内に**そのまま展開**され、**画面ロード/リロード時に必ず読み込まれる**。
- 用途：**関数（ファンクション）の定義置き場**、画面描画時のスタイル・値の初期設定。
- → 共通部品を呼ぶ「入口」になるのは基本ここ。

### (B) 各ボタンの「Click時JavaScript」/ FOCUS OUT
- ボタンの `onclick=""` 属性の**中に1行として展開**される。
- 注意点（マニュアル明記）：
  - `onclick` 内に1行展開されるので **`//` の行コメントは使用不可**（以降が全部コメント化する）。
  - 処理の最後に **`return false;` を書くとSubmit処理を中断**できる（バリデーションNG時に有効）。
- FOCUS OUT：テキスト/リストは blur、ラジオ/チェックは change イベントで発火。

## 4. TALON標準で用意されているCSJS関数・仕組み

CSJSから画面を操作するための、TALON独自の関数・変数がある（コードサンプル章より）。

| 仕組み | 内容 |
|---|---|
| **バインド変数** `%ブロック番号_物理項目名%` | 画面項目の値をJSで参照。例：`var nm = %1_ITEM_NM%;` |
| `getElementByName(name)` | 画面項目オブジェクトを取得（検索条件は `CNDTN_` 接頭辞、単票はブロック番号接頭辞） |
| `getValueByElement(...)` | 画面項目の「値」を取得 |
| `getButtonElement(...)` | ボタン要素を取得 |
| `resizeContents_start() / _end()` | 画面の再描画（初期表示・検索後など）の**前/後**に処理を実行。引数でイベント名・対象ボタン・行番号も取得可（Ver.6.0.2〜） |
| `onCloseMassageDialog()` | 確定後メッセージを閉じたときの処理 |
| `selectItem_start() / _end()` | 検索子画面の値を親画面へ渡すときの処理 |
| `document.getElementById('EVENT_STATUS')` 等 | 画面の状態・処理結果・機能IDを取得 |

> 注意：これらTALON独自のオブジェクトIDや命名規則は**バージョンアップで変わる可能性があり、下位互換保証の対象外**（マニュアル明記）。アップデート時は動作確認が必要。

## 5. CSJSを「共通化」する2つの公式の方法 ★今回の本題

共通部品（validation.js など）をどう全画面で使えるようにするか。公式に2通りある。

### 方法① talon_project.js（Docrootに置く・全画面で自動読込）
- **置き場所**：Payaraの docroot フォルダ
  - Ver.5 / 6.0：`C:\payara5\glassfish\domains\production\docroot`
  - Ver.6.2：`C:\payara6\glassfish\domains\production\docroot`
- **ファイル名**：`talon_project.js`（固定）
- 画面呼出時に自動で読み込まれる。**共通CSJSがある場合の標準的な配置方法。**
- 反映されないときはブラウザのキャッシュクリア。

→ 今回の5ファイルは、まとめて（または連結して）`talon_project.js` として置くのが素直。

### 方法② 共通Client-side JavaScriptメンテナンス（汎用コードマスタ登録・Ver.5.1.39〜）
- メニュー ＞ 機能構築 ＞ **共通Client-side JavaScriptメンテナンス** で登録。
- 設定項目：共通JSタイトル（ID）/ 共通JS名称 / JavaScriptソース。
- 登録すると、各機能のCSJS記述箇所で**「ライブラリ」としてチェックして選択**できる。
  - デフォルト全ON。**機能に必要なライブラリだけONにするのが推奨。**
- 登録内容は汎用コードマスタ（識別コード `TLN_GENERAL_CLIENT_JAVASCRIPT`）に保存され、**再ログインで反映**。

→ 「機能ごとに使う部品を選びたい」「画面ごとに読込を絞りたい」場合はこちら。

> 補足：JavaScriptエンジン（サーバ側）側の共通化は別物で「共通JavaScriptメンテナンス」を使う。CSJSとは混同しないこと。

### ①と②の使い分け

| | talon_project.js | 共通CSJSメンテナンス |
|---|---|---|
| 読み込み | 全画面で常に自動 | 機能ごとに選択 |
| 管理 | ファイル（Git管理しやすい） | DB（汎用コードマスタ） |
| 反映 | キャッシュクリア | 再ログイン |
| 向くケース | 全画面共通の土台部品 | 機能限定・読込を絞りたい部品 |

## 6. デバッグ方法（参考）
- ブラウザの開発者ツール（F12）。TALON実行画面のHTML `<script>` 末尾に展開されたコードが見える。
- Ver.6はSPA対応で、画面ロード時のソースにブレイクポイントを置いてもブレイクしないことがある。
  → コードに **`debugger;`** を書くと、検索などのイベント時に停止する。
- `console.log` も有効。

---

## まとめ（一言で）
**CSJS＝「DBに触らない、画面操作専門の補助JavaScript」**。
標準機能で作れない画面の振る舞い（チェック・計算・表示制御・確認）を足すために使う。
共通化は `talon_project.js`（全画面自動）か「共通CSJSメンテナンス」（機能別選択）の2択。
